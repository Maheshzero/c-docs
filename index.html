    🎓 Linux & Sandbox Simulation Playground

    This workspace operates as a full-featured mock Linux terminal and code editor. You can write custom programs, compile them, and view their execution outputs directly in the terminal!

    📂 Virtual Filesystem Commands

    • ls: Lists all C source files and compiled binaries (highlighted in green).
    • gedit <filename>.c: Opens the file in Gedit. If the file does not exist, a new empty file is created.
    • cat <filename>: Outputs the raw text content of the file directly to the terminal.
    • rm <filename>: Deletes the file from the virtual filesystem.

    ⚙️ Compilation & Execution

    • gcc <filename>.c -o a: Compiles the source file into an output binary named a.
    • ./a: Executes the compiled binary inside the terminal shell.
    • Dirty State Tracking: If you edit code, Gedit shows a modified indicator (*). Compiling a file while it has unsaved changes warns you that GCC is compiling the previously saved version on disk!

    🧪 Generic Code Sandbox

    When you compile a custom file (e.g. sandbox.c), running ./a dynamically scans your code. If it detects POSIX operations like fork(), shared memory, message queues, or multithreading, it simulates their runtime executions dynamically!

    ⚠️ Compiler Validations

    The compiler checks for common pitfalls: missing semicolons (;) throw syntax errors, and missing headers (e.g. <unistd.h> for fork or <sys/shm.h> for shared memory) trigger implicit declaration warnings.
